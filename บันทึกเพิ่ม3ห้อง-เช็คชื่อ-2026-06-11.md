## 3. วิธีนำขึ้นใช้งาน (Deploy)

1. เปิดโปรเจกต์ Apps Script "เว็บเช็คชื่อ"
2. ไฟล์ `index` (HTML) → ลบของเดิม → วางเนื้อหาจาก **index-appscript.html**
3. ไฟล์ `Code.gs` → ลบของเดิม → วางเนื้อหาจาก **google-apps-script.gs**
4. **Deploy → Manage deployments → ✏️ (ดินสอ) → Version: New version → Deploy**
   - ⚠️ ห้ามกด "New deployment" เพราะ URL จะเปลี่ยน
   - URL เดิมใช้ได้เหมือนเดิม

> ชีตของห้อง 4/1 และ 5/1 จะถูกสร้างให้อัตโนมัติเมื่อบันทึกครั้งแรก ไม่ต้องสร้างเอง

## 4. การแยกข้อมูลแต่ละห้อง (สำคัญ)

- ทุกครั้งที่เรียก API จะแนบพารามิเตอร์ `room` ไปด้วย (ฟังก์ชัน `sheetsFetch` ใส่ `currentRoom` ให้อัตโนมัติ)
- Apps Script จับคู่ห้อง → ชีต ดังนี้:

| ห้อง | ชีตข้อมูล | ชีตรูป |
|------|-----------|--------|
| ม.6/1 | `Attendance` (ชีตเดิม) | `Photos` (ชีตเดิม) |
| ม.4/1 | `Attendance_4_1` | `Photos_4_1` |
| ม.5/1 | `Attendance_5_1` | `Photos_5_1` |

- ห้อง 6/1 ใช้ชีตเดิม → **ข้อมูลเก่าที่บันทึกไว้ไม่หาย**
- localStorage แยก prefix ตามห้อง: `att4_1_`, `att5_1_`, `att6_1_` (และ `photo{ห้อง}_`)

---

## 5. จุดในโค้ดที่ควรรู้ (ไว้แก้ภายหลัง)

- `const ROOMS = {...}` — รายชื่อ + ชื่อครู + label ของแต่ละห้อง (key ต้องตรงกับ Apps Script: `4_1`/`5_1`/`6_1`)
- `const SAVE_PASSWORD = '1111'` — เปลี่ยนรหัสบันทึกตรงนี้
- `selectRoom(key)` — ทำงานตอนคลิกเลือกห้อง (เซ็ตห้อง + รายชื่อ + prefix)
- `changeRoom()` — ปุ่ม "🔄 เปลี่ยนห้อง" บน header
- ฝั่ง Apps Script: `normRoom`, `sheetNameFor`, `photoSheetNameFor`, `getSheet(room)`, `getPhotoSheet(room)`
