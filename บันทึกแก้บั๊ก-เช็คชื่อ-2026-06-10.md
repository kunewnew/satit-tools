# บันทึกการแก้บั๊กเว็บเช็คชื่อ ม.6/1 — 10 มิ.ย. 2569

## อาการ

กดบันทึกในเว็บแล้วขึ้น "บันทึกแล้ว" แต่เปิด Google Sheets ไม่เห็นข้อมูล

## สาเหตุ (มี 2 ปัญหาซ้อนกัน)

### 1. เปิดเว็บจาก URL เก่าที่ถูก archive

เคยกด **Deploy → New deployment** (แทนที่จะอัปเดตเวอร์ชันบน deployment เดิม) ทำให้ได้ URL ใหม่ ส่วน URL เก่า (`AKfycbw5yhrG...`) ถูกเก็บเข้าคลัง (archived) — เว็บ/bookmark ที่เปิดอยู่ยังชี้ไป URL เก่า

### 2. Sheets แปลงวันที่เป็น Date object

เซลล์คอลัมน์ A ที่เว็บเขียนค่า `2026-06-09` ลงไป ถูก Google Sheets แปลงเป็น Date โดยอัตโนมัติ ส่งผลให้

- `findDateRow()` เทียบ string `"2026-06-10"` กับ Date object ไม่เจอ → บันทึกซ้ำเป็น**แถวใหม่ทุกครั้ง** (เกิดแถว 10 มิ.ย. ซ้ำ 2 แถว)
- `getAll` คืน key เป็น `"Tue Jun 09 2026 00:00:00 GMT+0700..."` แทน `"2026-06-09"` → หน้าเว็บอ่านข้อมูลกลับมาไม่เจอ

## การแก้ไข

1. **แก้ `Code.gs`** (ไฟล์ local: `google-apps-script.gs` อัปเดตตรงกันแล้ว)
   - เพิ่ม `normDate(v)` — แปลงค่าเซลล์เป็น string `yyyy-MM-dd` เสมอ (รองรับ Date object, timezone Asia/Bangkok)
   - `findDateRow()` และ `getAll` ใช้ `normDate()` ในการเทียบ/คืนค่า
   - `getSheet()` บังคับคอลัมน์ A เป็น Plain text (`setNumberFormat('@')`) ทุกครั้ง กันการแปลงในอนาคต
2. **Deploy เวอร์ชัน 4** บน deployment เดิม (URL ไม่เปลี่ยน)
3. **ล้างข้อมูลใน sheet "Attendance"**
   - ลบแถว `2026-06-10` ที่ซ้ำ (เก็บอันล่าสุด 20:39)
   - ลบแถวทดสอบ `2099-01-01`
   - พิมพ์วันที่ 2 เซลล์ใหม่เป็น text (นำหน้าด้วย `'`)
4. **อัปเดต `DEFAULT_SCRIPT_URL`** ในไฟล์ `เช็คชื่อ-เตรียมวิศวะ.html` เป็น URL ใหม่

## ผลการทดสอบ

`?action=getAll` คืนค่า key `2026-06-09` และ `2026-06-10` ครบ 36 คน ✅
`?action=save` บันทึกลงแถวเดิม ไม่สร้างแถวซ้ำ ✅

## ข้อควรจำตอนแก้โค้ดครั้งหน้า

1. แก้โค้ดใน Apps Script editor → บันทึก (⌘S)
2. กด **Deploy → Manage deployments → ✏️ (แก้ไข) → Version: New version → Deploy**
3. **ห้ามกด "New deployment"** — จะได้ URL ใหม่ ทำให้ลิงก์เดิมใช้ไม่ได้อีก
